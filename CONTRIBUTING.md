# Contributing

Thank you for considering contributing.

## How to contribute

1. Open an issue to propose a change (bug, enhancement, documentation).
2. Fork the repository and create a branch:
   - `feature/<short-name>` or `fix/<short-name>`
3. Keep changes small and focused.
4. Test locally (see README: local server).
5. Open a Pull Request and describe:
   - what changed,
   - why it matters,
   - how to test.

## Development notes

- This project is a static web application (HTML/CSS/JS + CSV/JSON data).
- Avoid adding build tooling unless necessary.
- Prefer readable, well-commented JavaScript and accessible UI patterns.

## Reporting sensitive issues

If you discover a security or privacy issue, please follow `SECURITY.md`.
